* 널 참조
  * **컴퓨터에서 가장 많은 버그**를 차지 하는것
* Nullable
  * .kt
    ```kotlin
    val x: int = 3
* non-Nullable
  * .kt
    ```kotlin
    val x: int? = 3
* nullable(자식)과 non-nullable(부모)은 관계이다.
  * 그래서 다음과 같은 코드는 잘못된 코드이다.
    * .kt
      ```kotlin
      val x: Int? = 3
      val y: Int = x // 자식을 부모에 넣을수 없다.
